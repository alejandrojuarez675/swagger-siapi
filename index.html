Errores

La API usa los siguientes HTTP Status Codes:

HTTP Status Code Descripción Nota
200 Ok
202 Solicitud recibida En proceso (para servicios asincrónicos)
400 Error de datos Validación sobre el body o query string
401 No autorizado Credenciales inválidas
403 Acceso prohibido No posee permisos
404 Recurso no encontrado Incluye errores de validación de path params
500 Error en el servidor Error interno del servidor o del core de la aseguradora

La estructura de los errores será de la siguiente manera:

{
  "errores": [
    {
      "codigo": "PAGINA_NO_ENCONTRADA",
      "mensaje": "La URL que usted busca es inexistente"
    }
  ],
  "httpStatusCode": 404
}

Paths

Objetos asegurables

GET /objetos

Tags: Cotización

Los objetos representan la cosa el bien que se está buscando asegurar. En el mundo de los seguros también se lo suele llamar “bien asegurable”.

200 OK

Retorna el listado de objetos.

500 Internal Server Error

Error interno del servidor

Indicios asegurables

GET /objetos/{Objeto}/indicios

Tags: Cotización

Los indicios representan eventos que puede sufrir un objeto y para cuya ocurrencia se puede contratar un seguro.

Formulario interactivo

POST /objetos/{Objeto}/indicios/{Indicio}/formulario/proxima-pregunta

Tags: Cotización

Retorna un listado de preguntas y alertas filtrado en función de las respuestas previas indicadas.

Si se lo invoca con un body request vacío, generará un nuevo consultaId y lo retornará, en conjunto con las preguntas iniciales a hacer.

Código del objeto

Indicio

200 OK

Retorna un listado de preguntas y alertas filtrado según las respuestas previas. Una lista vacía indica que ya no quedan preguntas por realizar.
